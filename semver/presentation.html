What is Semantic Versioning?
MAJOR.MINOR.PATCH
Increment MAJOR:
  • Incompatible API changes
Increment MINOR:
  • New Features
  • Backwards Compatible
Increment PATCH:
  • Bugfixes ONLY
  • Backwards Compatible
Numbering:
  • 0.3.1 → 0.3.2   (PATCH)
  • 0.3.2 → 0.4.0   (MINOR)
  • 0.4.0 → 0.4.1   (PATCH)
  • 0.4.1 → 1.0.0   (MAJOR)
More Info: semver.org
Why is Semantic Versioning Important?
Communicate intentions to your users

Will this break my app that has dependencies on it? Will I receive any new functionality for upgrading?

Avoid Dependency Hell
Standards → Consistency → Trust
